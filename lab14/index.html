Challenge

The challenge was to learn how to debug our programs!

Thoughts

My interactions with the console while at various debugger breakpoints!

Flexbox is so, so cool... making those funny caption boxes that I seem to love doing so much is waaay easy now!

For this lab, in addition to doing the debugging stuff detailed below, I also experimented with using "debugger" to create a breakpoint for a test function. I've worked with breakpoints before, so it works about how I expected to, but it was still really neat to see it in action! MDN's documentation , as always, helped out here. The coolest part about this to me was that I could still interact with the console while we were paused at the breakpoint! While I'm used to setting watches and seeing the changes passively, the debugger here lets me interact with the page at that point in execution live! Super cool!!!

Results

Debugging!

I didn't really have too much to debug, since for the most part the script portions of my labs are the parts I generally do well on! Really, if this was about the CSS of my early labs instead...

However! I did find that a lab I completed to the assignment's specifications - Lab 6 - didn't seem to work the way I expected, and wouldn't print out the age property of my car object. I found out later that it is probably because of the way JSON.stringify() turns data from arrays and objects into strings. According to the MDN documentation for JSON.stringify() , the "replacer" function used has this default behavior:

"If [your key returns] a Function, Symbol, or undefined, the property is not included in the output."

So, I simply wrote a new replacer! Now my car's age gets stringify-d with the rest of the properties!
